🚀 Getting Started

Set up StoatMod in minutes

Follow this guide to invite the bot, configure your environment, and get it running on Railway or your own server.

Step 1 — Add StoatMod to your server

Click the button below to open the official Stoat bot invite page. You must have Manage Server permission on the target server.

ℹ️ StoatMod uses only the permissions it needs: sending messages, managing roles (for verification), and reading message history (for moderation logging). It never requests admin.
Add StoatMod to Stoat

Required Permissions

When Stoat shows the permissions screen, ensure the following are granted:

  • Read Messages / View Channels
  • Send Messages & Embeds
  • Manage Messages (for automod deletion)
  • Manage Roles (for verification & reaction roles)
  • Kick & Ban Members (for moderation commands)
  • Read Message History

What is the invite URL format?

Stoat.chat uses this invite URL format (already set up in the button above):

https://stoat.chat/bot-invite?bot_id=YOUR_BOT_CLIENT_ID

Replace YOUR_BOT_CLIENT_ID with the Client ID from your bot's settings page at stoat.chat/settings/bots.

Step 1 of 5

Step 2 — Get your bot credentials from Stoat

You need two values from the Stoat developer portal: your Bot Token and your Client ID (Bot ID).

  1. Go to stoat.chat/settings/bots and open your bot.
  2. Copy the Bot Token — keep this secret! Never share it or commit it to git.
  3. Copy the Bot ID / Client ID — this is public-safe and used in the invite URL.
  4. Also copy your own User ID from Stoat (run !myid in any server) — this becomes the BOT_OWNER_ID.
⚠️ Never paste your bot token into this website or any public form. The token is entered only in your private .env file or your hosting platform's secret variables.

Supabase database setup

StoatMod uses Supabase for all persistent data (server settings, user XP, moderation logs, etc.).

  1. Create a free project at supabase.com.
  2. Go to Settings → API. You need the Project URL, the anon key (public, safe for frontend), and the service_role key (secret — backend only).
  3. Run the database migrations provided in apps/logiq-bot/scripts/ against your Supabase project to create the required tables.
Step 2 of 5

Step 3 — Configure environment variables

Copy the template below into a file called .env inside apps/logiq-bot/ and fill in your values. On Railway/VPS, add these as environment secrets instead.

# ── Stoat Bot ───────────────────────────────────── STOAT_BOT_TOKEN=your_bot_token_here # secret — never expose STOAT_BOT_ID=your_bot_id_here # public-safe client ID BOT_OWNER_ID=your_stoat_user_id # your own Stoat user ID STOAT_TEST_CHANNEL_ID= # optional startup test channel # ── Supabase ─────────────────────────────────────── SUPABASE_URL=https://xxxx.supabase.co SUPABASE_SERVICE_KEY=your_service_role_key # secret SUPABASE_ANON_KEY=your_anon_key # ── Optional AI / Social features ───────────────── OPENAI_API_KEY= ANTHROPIC_API_KEY= TWITCH_CLIENT_ID= TWITCH_CLIENT_SECRET= YOUTUBE_API_KEY= # ── Environment ──────────────────────────────────── ENVIRONMENT=production

✅ Add .env to your .gitignore — it should never be committed. The repo already includes .env.example as a safe template.
Step 3 of 5

Step 4 — Deploy the bot

Choose your preferred deployment method:

  1. Go to railway.app and create a new project from your GitHub repo.
  2. Set the Root Directory to apps/logiq-bot in Railway's service settings.
  3. Add all required environment variables in Railway → Variables (same list from Step 3).
  4. Railway will detect requirements.txt and runtime.txt and build automatically.
  5. The start command is auto-read from railway.json: python main.py
📡 The bot runs a health server on port 8080. Railway will automatically expose this for uptime monitoring. You can use it in the Status page too.
# railway.json (already in repo — no changes needed) { "deploy": { "startCommand": "python main.py", "healthcheckPath": "/health", "healthcheckTimeout": 30 } }
# From repo root: docker compose up stoatmod-bot -d

Or manually:

cd apps/logiq-bot docker build -t stoatmod-bot . docker run -d --env-file .env -p 8080:8080 stoatmod-bot
📦 A compose.yml is available at the monorepo root that starts both the bot and a local Redis (if needed). Edit the service name if required.
  1. SSH into your VPS and clone the repo.
  2. Install Python 3.11+: apt install python3.11 python3-pip
  3. Install dependencies: pip install -r requirements.txt
  4. Copy .env.example to .env and fill in your values.
  5. Run with systemd or screen: python main.py
# Example systemd unit — /etc/systemd/system/stoatmod.service [Unit] Description=StoatMod Bot After=network.target [Service] WorkingDirectory=/opt/stoatmod/apps/logiq-bot EnvironmentFile=/opt/stoatmod/apps/logiq-bot/.env ExecStart=/usr/bin/python3 main.py Restart=always RestartSec=5 [Install] WantedBy=multi-user.target
Step 4 of 5
🎉

StoatMod is live!

Your bot should now be online in your server. You'll see a startup message in the test channel (if configured) and the bot will respond to !help.

Quick setup commands

Once the bot is in your server, run these commands to finish setup:

CommandDescription
!helpList all available commands
!setupInteractive server configuration wizard
!prefix <char>Change the command prefix (default: !)
!welcome channel #chanSet the welcome message channel
!modlog channel #chanSet the mod-action log channel
!verify setupEnable member verification
