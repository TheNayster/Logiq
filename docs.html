Setup Guide Bot Status

🛡️ Moderation

Core moderation commands. All actions are logged to your configured mod-log channel and stored in Supabase.

CommandArgsDescriptionPermission
!warn@user [reason]Issue a warning. Auto-punishes at max_warnings threshold (config.yaml).Mod
!warnings@userList a user's active warnings.Mod
!clearwarn@user [id]Remove a specific or all warnings from a user.Mod
!mute@user [duration] [reason]Timeout a user. Duration: 10m, 1h, 1d, etc.Mod
!unmute@userRemove a timeout from a user.Mod
!kick@user [reason]Kick a member from the server.Mod
!ban@user [days] [reason]Ban a member. Optionally purge N days of messages.Mod
!unbanuser_idRevoke a ban by user ID.Mod
!purge[count]Bulk-delete up to 100 recent messages in the channel.Mod
!modlogchannel #chanSet the channel where mod actions are posted.Admin

📈 Leveling & XP

Members earn XP for every message (with a per-user cooldown to prevent spam farming). Configure XP rates in config.yaml → modules.leveling.

CommandArgsDescription
!rank[@user]Show your (or another user's) current level and XP.
!leaderboard[page]Top members by XP for this server.
!xp@user +/-NManually adjust a user's XP (mod only).
ℹ️XP is scoped per-server. The xp_cooldown (seconds) prevents XP farming from rapid messages. Tune xp_per_message and level_up_multiplier in config.yaml.

💰 Economy

Virtual currency system. Starting balance and daily rewards are configurable. Economy is in gradual rollout (50% of servers by default).

CommandArgsDescription
!balance[@user]Check your (or another user's) coin balance.
!dailyClaim your daily reward (once per 24h).
!give@user amountTransfer coins to another member.
!richlist[page]Economy leaderboard.

🎫 Tickets

Support ticket system. Max 3 open tickets per user by default. Tickets auto-close after 7 days of inactivity.

CommandArgsDescription
!ticket[reason]Open a new support ticket.
!ticket close[reason]Close the current ticket channel.
!ticket add@userAdd a user to the current ticket.
!ticket remove@userRemove a user from the current ticket.

✅ Verification

Gate server access behind a verification step. Supports DM or button-based verification.

CommandArgsDescription
!verify setupConfigure verification for this server (interactive).
!verify methoddm | buttonSwitch verification method.
!verify role@roleSet the role granted after successful verification.

📢 Social Alerts

Post automatic alerts when your Twitch, YouTube, or Twitter accounts post new content.

CommandArgsDescription
!alerts twitchchannel #chan usernamePost Twitch live alerts to a channel.
!alerts youtubechannel #chan yt_channel_idPost YouTube upload alerts.
!alerts twitterchannel #chan @handlePost new tweets to a channel (requires Twitter API key).
!alerts removeplatformRemove an alert subscription.
!alerts listShow all active alert subscriptions.
⚠️Social alerts require platform API keys set in your .env. Check the environment variables guide.

AI-powered chat commands using OpenAI or Anthropic. Set provider in config.yaml → modules.ai_chat.

CommandArgsDescription
!askquestionAsk the AI a question. Response is channel-public.
!aichat on/offToggle AI chat for this server (admin only).

config.yaml Reference

The main config file is at apps/logiq-bot/config.yaml. Environment variables referenced as ${VAR} are expanded at startup.

# Core bot settings bot: prefix: "!" case_insensitive: true # Stoat API endpoints stoat: api_base: "https://stoat.chat/api" ws_url: "wss://stoat.chat/socket" timeout: 30 # Feature module toggles & settings modules: moderation: enabled: true max_warnings: 3 # auto-action at this threshold leveling: enabled: true xp_per_message: 10 xp_cooldown: 60 # seconds between XP awards level_up_multiplier: 1.5 ai_chat: enabled: true provider: "openai" # or "anthropic" # Optional web dashboard API (enables /stats and /guilds) web: enabled: false host: "0.0.0.0" port: 8000 cors_origins: ["https://stoatmod.vercel.app"]
ℹ️Enable web.enabled: true to activate the FastAPI REST server on port 8000. The health check server always runs on port 8080 regardless.

Bot REST API

When web.enabled: true in config.yaml, the bot exposes a FastAPI server. The health server (always on) is on port 8080.

Health server (port 8080 — always on)

GET/health
Returns overall health and per-component status (bot, database, adapter, cogs). Used by Railway for uptime checks and by the status page.
GET/info
Returns bot metadata: version, platform, environment, uptime (seconds), and number of loaded cogs.

Web API server (port 8000 — when web.enabled: true)

GET/
Service info: name, version, platform.
GET/health
Same as health server /health. Useful if only one port is exposed.
GET/stats
Returns server and user counts from Supabase.
⚠️The web API has no authentication in the current implementation. Do not expose sensitive endpoints publicly. CORS origins are configurable in config.yaml → web.cors_origins.
